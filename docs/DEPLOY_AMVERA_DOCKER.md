# Деплой на Amvera Cloud: через Dockerfile

Вариант развёртывания бэкенда как **Docker-образа** на [cloud.amvera.ru](https://cloud.amvera.ru/projects). Фронт по-прежнему можно развернуть отдельно (Node или свой образ).

---

## Что получится

- **Бэкенд:** один образ на базе Python, в котором запускается только uvicorn (FastAPI). Telegram-бот работает по webhook: обновления приходят на тот же сервис на путь `/api/telegram/webhook`. Отдельного процесса для бота в контейнере нет.
- Сборка по **Dockerfile** в корне репозитория; образ включает `backend/`, `bot/` и зависимости из `requirements.txt`. Порт контейнера задаётся переменной **PORT** (Amvera подставляет при запуске).

---

## Подготовка

Подключите репозиторий к Amvera по Git. В корне должен быть Dockerfile, собирающий Python-приложение: установка системных зависимостей (при необходимости для asyncpg), копирование `requirements.txt`, установка пакетов, копирование кода `backend/` и `bot/`, запуск одной командой uvicorn с хостом `0.0.0.0` и портом из переменной окружения. Фронт для этого варианта деплоится как отдельное приложение (например, по инструкции с Node или своим Dockerfile).

---

## База данных и переменные

Создайте PostgreSQL в Amvera (или используйте свою). Строку подключения в формате `postgresql+asyncpg://USER:PASSWORD@HOST:PORT/DATABASE` задайте в переменных окружения приложения как **DATABASE_URL**. Таблицы создаются и обновляются при старте контейнера через Alembic (миграции выполняются в lifespan).

Также задайте: **TELEGRAM_BOT_TOKEN**, **FRONTEND_URL** (URL фронта после его деплоя), **SYSTEM_ADMIN_TELEGRAM_IDS**. **BACKEND_URL** должен быть равен **публичному URL этого приложения** (без слэша в конце) — по нему при старте приложения выставляется webhook бота в Telegram. При необходимости добавьте переменные ЮKassa и прочие параметры из настроек проекта.

---

## Создание приложения и сборка

В панели Amvera создайте приложение и выберите сборку **по Dockerfile** (не встроенное окружение Python). Укажите репозиторий и корень сборки (корень репозитория, где лежит Dockerfile). Платформа выполнит `docker build` и запустит контейнер; порт приложения задаётся платформой через **PORT**.

Убедитесь, что в образе не жёстко прописан порт 8000 без учёта **PORT** — иначе приложение может не открыться снаружи. В проекте команда запуска в Dockerfile использует переменную окружения порта.

---

## После деплоя

Проверьте: откройте `https://URL-ПРИЛОЖЕНИЯ/health` — должен вернуться ответ `{"status":"ok"}`. Проверьте, что **BACKEND_URL** в настройках совпадает с этим URL. В ЮKassa укажите webhook: `https://URL-ПРИЛОЖЕНИЯ/api/payments/yookassa/webhook`. URL приложения подставьте в конфиг фронта как адрес API и в BotFather как адрес Web App — в зависимости от того, как развёрнут фронт.

---

## Фронт

Для Mini App создайте отдельное приложение (Node по папке `frontend` или свой образ). В его переменных укажите **BACKEND_URL** равным URL этого бэкенда. В переменных бэкенда задайте **FRONTEND_URL** равным URL фронта.
