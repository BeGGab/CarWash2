# Деплой на Amvera Cloud: Python (бэкенд) и Node (фронт)

Развёртывание без Docker: бэкенд — встроенное окружение **Python**, фронт — **Node (npm)**. Подходит для [cloud.amvera.ru](https://cloud.amvera.ru/projects).

---

## Что получится

- **Приложение 1 (бэкенд):** один процесс — FastAPI (uvicorn). Telegram-бот получает обновления по webhook на этот же сервис (`/api/telegram/webhook`), отдельный процесс для бота не запускается.
- **Приложение 2 (фронт):** сборка Vite, при старте из переменной `BACKEND_URL` формируется `config.json`, статика раздаётся через `serve`.
- **База:** управляемый PostgreSQL в Amvera или своя; строка подключения задаётся в переменных бэкенда.

---

## Подготовка

Зарегистрируйтесь на [cloud.amvera.ru](https://cloud.amvera.ru/), подключите репозиторий по Git (свой или Git Amvera). В корне репозитория для бэкенда используется конфигурация сборки и запуска на Python (amvera.yaml); для фронта сборка и запуск задаются из папки `frontend` (там свой конфиг под Node). Убедитесь, что в репозитории есть папки `backend/`, `bot/`, `frontend/`, зависимости перечислены в `requirements.txt` и в `frontend/package.json`.

---

## База данных PostgreSQL

В панели Amvera создайте PostgreSQL (раздел баз данных). Сохраните хост, порт, имя базы, пользователя и пароль. Соберите строку подключения в формате `postgresql+asyncpg://USER:PASSWORD@HOST:PORT/DATABASE` — она понадобится для переменной `DATABASE_URL` бэкенда. Таблицы создаются и обновляются при старте приложения через Alembic (миграции выполняются в lifespan).

---

## Приложение бэкенда (Python)

Создайте новое приложение, выберите окружение **Python** (не Docker). Укажите корень сборки — **корень репозитория**. Платформа по конфигурации в корне установит зависимости из `requirements.txt` и запустит веб-процесс командой, указанной в секции запуска (uvicorn для FastAPI). Порт задаётся переменной **PORT**, которую подставляет Amvera.

В разделе переменных окружения задайте: `DATABASE_URL`, `TELEGRAM_BOT_TOKEN`, `FRONTEND_URL` (публичный URL фронта после его деплоя), `SYSTEM_ADMIN_TELEGRAM_IDS` (ID через запятую). **BACKEND_URL** должен быть равен **публичному URL этого приложения** (без слэша в конце) — по нему при старте регистрируется webhook бота. При необходимости добавьте переменные ЮKassa (`YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY`, `YOOKASSA_CURRENCY`), `CORS_EXTRA_ORIGINS`, `REFUND_HOURS_BEFORE_START` и др.

После деплоя сохраните URL приложения — он понадобится для фронта и для ЮKassa (webhook: `https://URL-БЭКЕНДА/api/payments/yookassa/webhook`).

## Приложение фронта (Node)

Фронт — это отдельное приложение в Amvera. Оно собирает проект из папки `frontend` и раздаёт готовые файлы как статический сайт.

### Шаг 1. Создать приложение

1. В панели [cloud.amvera.ru](https://cloud.amvera.ru/projects) нажмите **«Создать приложение»** (или аналог).
2. Подключите **тот же репозиторий**, что и для бэкенда (один репо — два приложения).
3. В качестве типа/окружения выберите **Node** (не Docker, не Python).

### Шаг 2. Указать корень сборки (Root / Build context)

Сейчас в репозитории код фронта лежит в папке **`frontend`**, а не в корне. Нужно сказать Amvera: «собирать приложение из папки `frontend`».

- В настройках приложения найдите поле вроде **«Корень сборки»**, **«Root directory»**, **«Build context»** или **«Монопольная директория»**.
- Укажите туда **`frontend`** (без слэша в начале, просто имя папки).

Что это даёт: при сборке платформа будет работать внутри папки `frontend`, там найдёт `package.json`, `amvera.yaml` и весь код. Если этого поля нет — в документации Amvera посмотрите, как задать подпапку репозитория для Node-приложения.

### Шаг 3. Сборка (Build)

Платформа должна выполнить:

1. **Установка зависимостей** — команда `npm ci` (или `npm install`) в папке `frontend`. Создаётся папка `node_modules`.
2. **Сборка проекта** — команда `npm run build`. Vite собирает приложение в папку `dist/`.

Если в проекте есть файл **`frontend/amvera.yaml`** с секцией `build`, Amvera может подхватить эти шаги автоматически. Иначе в разделе **«Сборка»** / **«Build»** укажите вручную:

- Команда установки: `npm ci` (или `npm install`).
- Команда сборки: `npm run build`.

В итоге после сборки в папке `frontend` должна появиться папка **`dist`** с готовыми файлами (HTML, JS, CSS).

### Шаг 4. Запуск (Run)

При запуске приложения платформа подставляет переменную **PORT** (номер порта, на котором нужно слушать). Нужно:

1. Записать в `dist/config.json` адрес бэкенда из переменной окружения **BACKEND_URL** (это делает скрипт `npm run start` через `write-config.cjs`).
2. Запустить раздачу статики из папки `dist` на порт **PORT**.

В проекте это уже сделано: команда запуска — **`npm run start`**. Она выполняет `node scripts/write-config.cjs` (пишет `config.json` из `BACKEND_URL`) и затем `npx serve -s dist -l $PORT` (раздаёт папку `dist` на порт из переменной PORT).

Если платформа читает **`frontend/amvera.yaml`**, там указано `run.web: npm run start`. Иначе в разделе **«Запуск»** / **«Run»** / **«Start command»** укажите вручную: **`npm run start`**.

### Шаг 5. Переменные окружения

В настройках **этого** (фронтового) приложения откройте **«Переменные окружения»** и добавьте одну переменную:

| Имя           | Значение |
|---------------|----------|
| **BACKEND_URL** | Публичный URL бэкенда (приложения 1), **без** слэша в конце. Например: `https://your-backend.amvera.cloud` |

По этому значению при старте формируется `dist/config.json` с полем `backendUrl` — фронт будет обращаться к вашему API по этому адресу.

### Шаг 6. Деплой и итог

Запустите сборку и деплой. После успешного запуска в панели будет указан **URL приложения** (например `https://your-frontend.amvera.cloud`) — это адрес вашего Mini App.

- В **BotFather** укажите этот URL как адрес кнопки Web App.
- В настройках **бэкенда** в переменных окружения задайте **FRONTEND_URL** равным этому же URL (для CORS и редиректов после оплаты).

## После деплоя

Проверьте бэкенд: откройте в браузере `https://URL-БЭКЕНДА/health` — ответ `{"status":"ok"}`. Убедитесь, что в настройках бэкенда **BACKEND_URL** совпадает с этим URL — иначе webhook бота зарегистрируется неверно. В личном кабинете ЮKassa укажите webhook: `https://URL-БЭКЕНДА/api/payments/yookassa/webhook`. Логи приложений смотрите в панели Amvera.
