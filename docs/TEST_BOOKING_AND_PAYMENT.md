# Бронирование и оплата в тестовом режиме (ЮKassa)

## Запуск с телефона (WebApp из Telegram)

Чтобы не было ошибки «Failed to fetch» при отправке геолокации и загрузке моек, клиент должен знать URL бэкенда. Положите рядом с `index.html` (в корень фронта при деплое) файл **config.json** с содержимым:
```json
{"backendUrl": "https://ВАШ_ПУБЛИЧНЫЙ_URL_БЭКЕНДА"}
```
Пример: `frontend/public/config.example.json`. При сборке (`npm run build`) файл из `public/` попадёт в корень `dist/` — скопируйте его как `config.json` и подставьте свой URL бэкенда.

## Что нужно для теста

1. **Аккаунт ЮKassa** и включённый тестовый режим в личном кабинете.
2. В `.env` указаны тестовые данные:
   - `YOOKASSA_SHOP_ID` — ID магазина из ЛК ЮKassa
   - `YOOKASSA_SECRET_KEY` — тестовый секретный ключ (в тестовом режиме выдают тестовые ключи)
   - `YOOKASSA_CURRENCY=RUB`
3. **Webhook** в ЛК ЮKassa: URL вашего бэкенда, например:
   - Локально через туннель: `https://ВАШ_ТУННЕЛЬ.ngrok.io/api/payments/yookassa/webhook`
   - На сервере: `https://ваш-домен.ru/api/payments/yookassa/webhook`
4. В `.env` бэкенда указан **публичный** `FRONTEND_URL` (куда ЮKassa вернёт пользователя после оплаты), например:
   - `FRONTEND_URL=https://ваш-webapp.serveo.net`
   - В тестовом режиме возврат идёт на `FRONTEND_URL/payment-success?booking_id=...`

## Пошагово: довести бронь до конца и оплатить

### 1. Подготовка данных

- Пользователь должен быть зарегистрирован (через бота: `/start` и отправить контакт, либо через API `POST /api/auth/register` с `telegram_id`).
- В системе есть хотя бы одна **одобренная** мойка (системный админ подтвердил её в «Мойки на модерации»).
- У мойки есть хотя бы одна услуга (админ мойки добавил в «Кабинет автомойки»).

### 2. Действия в Mini App (клиент)

1. Открыть Mini App из Telegram (кнопка «Открыть мини-приложение» в боте).
2. Разрешить геолокацию или нажать «Показать моики в [город]».
3. Выбрать мойку из списка.
4. Выбрать **услугу**, **дату** и **время** (слот).
5. Нажать «Продолжить».
6. На экране подтверждения нажать **«Оплатить X ₽»** (предоплата 50%).
7. Произойдёт редирект на страницу оплаты ЮKassa (в тестовом режиме — тестовая форма).

### 3. Оплата в тестовом режиме ЮKassa

На странице ЮKassa использовать **тестовые карты** (из документации ЮKassa):

- **Успешная оплата:**  
  Номер: `1111 1111 1111 1026`  
  Срок: любой будущий (например 12/24), CVC: любые 3 цифры.

- **Отклонённая оплата:**  
  Номер: `1111 1111 1111 1034`

После успешной оплаты ЮKassa перенаправит на:
`FRONTEND_URL/payment-success?booking_id=<id>`

В Mini App откроется экран «Оплата прошла!» и можно перейти в «Мои брони» или открыть бронирование по кнопке.

### 4. Что делает бэкенд

- ЮKassa отправляет webhook `payment.succeeded` на `POST /api/payments/yookassa/webhook`.
- Бэкенд обновляет статус платежа и переводит бронь в статус **PAID**.
- Пользователь видит бронь в «Мои брони» с QR-кодом для предъявления на мойке.

### 5. Частые проблемы

| Проблема | Решение |
|----------|--------|
| «Failed to fetch» при запросе к API с телефона | Сборка фронта с `VITE_BACKEND_URL=https://ВАШ_ПУБЛИЧНЫЙ_БЭКЕНД`. Бэкенд и фронт должны быть доступны по HTTPS с телефона; в CORS добавлен origin вашего WebApp. |
| После оплаты не возвращает в приложение | Проверить `FRONTEND_URL` в `.env` (без слэша в конце). В настройках платежа ЮKassa возврат идёт на `FRONTEND_URL/payment-success?booking_id=...`. |
| Webhook не приходит, статус брони не меняется | Указать в ЛК ЮKassa URL webhook на публичный адрес бэкенда; убедиться, что туннель/сервер доступен с интернета. |
| Нет тестовых ключей | В личном кабинете ЮKassa включить «Тестовый режим» и взять тестовые Shop ID и секретный ключ. |

## Краткий чеклист

1. ✅ В ЮKassa включён тестовый режим, в `.env` — тестовые `YOOKASSA_SHOP_ID` и `YOOKASSA_SECRET_KEY`.
2. ✅ В ЮKassa настроен webhook на `https://ваш-бэкенд/api/payments/yookassa/webhook`.
3. ✅ В `.env` задан публичный `FRONTEND_URL` (адрес Mini App).
4. ✅ Фронт собран с `VITE_BACKEND_URL=https://ваш-бэкенд` (или раздаётся с того же домена, что и API).
5. ✅ Есть одобренная мойка с услугами и пользователь зарегистрирован.
6. ✅ В Mini App: мойка → услуга → дата/время → Оплатить → на странице ЮKassa ввести тестовую карту `1111 1111 1111 1026` → после редиректа бронь в статусе «Подтверждено» и с QR.
