# Развёртывание контейнера для фронтенда (WebApp)

Инструкция по сборке и запуску **Docker-контейнера** с клиентским приложением CarWash (Vite + React): мини-приложение для пользователей и админ-панели.

---

## Что входит в образ

- Собранная статика (Vite `npm run build`) — SPA и точки входа для админок.
- **Nginx** — раздача файлов на порту 80, поддержка SPA-маршрутов и `/config.json`.
- **config.json** формируется при старте контейнера из переменной окружения **`BACKEND_URL`** (URL вашего API). Без неё приложение не сможет вызывать бэкенд с телефона.

После развёртывания контейнера полученный **внешний URL фронта** нужно указать в Telegram (кнопка Web App) и в бэкенде в переменной **`FRONTEND_URL`** (CORS и редирект после оплаты).

---

## 1. Файлы в репозитории

В папке **`frontend/`** используются:

| Файл | Назначение |
|------|------------|
| `Dockerfile` | Двухэтапная сборка: Node — сборка, nginx — раздача статики. |
| `nginx.conf` | Конфиг nginx: корень SPA, маршруты админок, кэш. |
| `docker-entrypoint.sh` | Перед запуском nginx записывает `config.json` из `BACKEND_URL`. |

---

## 2. Сборка образа

**Из корня проекта** (контекст — папка `frontend/`):

```bash
docker build -f frontend/Dockerfile -t carwash-frontend frontend
```

**Из папки `frontend/`:**

```bash
cd frontend
docker build -t carwash-frontend .
```

---

## 3. Запуск контейнера

### Обязательная переменная

- **`BACKEND_URL`** — публичный URL бэкенда (API). По нему приложение будет ходить за данными (мойки, брони, оплата и т.д.).

**Пример запуска:**

```bash
docker run -d \
  --name carwash-frontend \
  -p 8080:80 \
  -e BACKEND_URL=https://your-backend.amverum.cloud \
  carwash-frontend
```

Фронт будет доступен по адресу `http://localhost:8080` (или по внешнему адресу вашего сервера, если порт проброшен).

### Проверка

1. В браузере: `http://localhost:8080` — должна открыться главная страница приложения.
2. Откройте `http://localhost:8080/config.json` — в ответе должен быть JSON с полем `backendUrl`, равным заданному `BACKEND_URL`.

## 4. Деплой на Amverum Cloud (пошагово)

Фронт разворачивается **отдельным приложением** в [Amverum Cloud](https://amverum.com/). Нужен уже задеплоенный бэкенд и его публичный URL.

### Шаг 1. Вход в панель

1. Зайдите на [cloud.amvera.ru](https://cloud.amvera.ru/).
2. Создайте **новое приложение** (отдельно от приложения с бэкендом и ботом).

### Шаг 2. Репозиторий

Подключите тот же репозиторий, что и для бэкенда (или загрузите код с папкой `frontend/`).

### Шаг 3. Сборка (Docker)

**Вариант A — контекст = папка `frontend` (предпочтительно)**

- **Контекст сборки:** папка **`frontend`** (или путь `./frontend`).
- **Dockerfile:** **`Dockerfile`** (файл `frontend/Dockerfile`).

**Вариант B — контекст = корень репозитория**

Если в Amverum нельзя выбрать подпапку как контекст:

- **Контекст сборки:** корень репозитория.
- **Dockerfile:** **`Dockerfile.frontend`** (файл в корне проекта).

### Шаг 4. Переменные окружения

В настройках приложения добавьте одну переменную:

| Переменная     | Значение |
|----------------|----------|
| **BACKEND_URL** | Публичный URL бэкенда, без слэша в конце. Например: `https://carwash-api-xxx.amverum.cloud` |

### Шаг 5. Деплой

Запустите сборку и деплой. После успешного запуска скопируйте **URL приложения** (например, `https://carwash-web-xxx.amverum.cloud`) — это URL вашего WebApp.

### Шаг 6. Настройка после деплоя

1. **Telegram (BotFather):** URL кнопки Web App = URL фронта (из шага 5).
2. **Бэкенд:** в переменных окружения приложения бэкенда задайте **`FRONTEND_URL`** = этот же URL (CORS и редирект после оплаты).
3. **ЮKassa:** возврат после оплаты — на этот URL, например `https://ваш-url/payment-success`.

Подробнее про бэкенд и общую схему см. [docs/DEPLOY_AMVERA_PYTHON_NODE.md](DEPLOY_AMVERA_PYTHON_NODE.md) и [docs/DEPLOY_AMVERA_DOCKER.md](DEPLOY_AMVERA_DOCKER.md).

---

## 5. Деплой на другую облачную платформу

1. Соберите образ (раздел 2) или настройте сборку из Git (Dockerfile в `frontend/` или `Dockerfile.frontend` в корне).
2. Задайте переменную **`BACKEND_URL`** = публичный URL бэкенда.
3. Контейнер слушает порт **80**. После деплоя возьмите URL приложения и укажите его в Telegram и в `FRONTEND_URL` на бэкенде.

---

## 6. Переменные окружения контейнера

| Переменная | Обязательна | Описание |
|------------|-------------|----------|
| `BACKEND_URL` | Да | Публичный URL API (бэкенда). Записывается в `/config.json` при старте. |

Других переменных для работы фронта не требуется.
